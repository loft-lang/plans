# loft-lang · plans

**The single overview of what the loft ecosystem is building.**

Every plan across the ecosystem — the **loft** language, the **libraries**, the
engine (**lavition**), the **games** — is tracked here as one issue, so the whole
roadmap reads in one place instead of scattered across repos.

- **Read it by subject.** Each title leads with **`[loft]` / `[libs]` /
  `[audience]` / …** — the primary cut: *what kind of thing this serves*. The
  `status:*` labels (active · future · deferred · finished) carry the lifecycle.
- **Design lives in each code repo** (its `plans/` dir). The issue here is the
  *tracking* + the overview, and links to its design.
- **`@PLN<N>` is the issue number** — `@PLN5` = `#5` (`…/issues/5`), the canonical
  reference. GitHub already shows it as `#N`, so the title doesn't repeat it.
